# Análisis de Necesidades {#análisis-de-necesidades}

Dentro de la asignatura de Sistemas Informáticos, hemos llevado a cabo varias actividades en las que hemos necesitado algunas herramientas que nos hacen la tarea más sencilla. En las últimas semanas, dos de estas herramientas han destacado por su utilidad para trabajar con proyectos Open Source. 

Estas herramientas son Docker y Apache Guacamole, dos soluciones de OpenSource y muy útiles para usuarios sin mucha experiencia. 

## Descripción de las herramientas {#descripción-de-las-herramientas}

La primera de ellas, Docker, se aplica a conceptos como empaquetar aplicaciones y ejecutarlas en contenedores parecidos a una máquina virtual, pero más simplificadas. Estos contenedores protegen las aplicaciones  de las diferencias de entorno existentes en diferentes sistemas operativos o máquinas virtuales. En nuestro caso, hemos desarrollado desde el entorno Visual Code, desde donde hemos desplegado remotamente las tareas que se nos especifican.  \[1\]

Por otro lado, Apache Guacamole sirve como un portal de escritorio remoto accesible a través del navegador web. Este también es de código abierto, permitiendo mayor accesibilidad por parte de cualquier usuario. El uso que le damos es tener una referencia gráfica del trabajo que estamos llevando a cabo.

### ¿Por qué elegimos estas soluciones? {#¿por-qué-elegimos-estas-soluciones?}

Con estas dos aplicaciones, se permite un trabajo paralelo bien sincronizado, sin depender de herramientas externas y haciendo uso de entornos como VSCode o el escritorio remoto a través del navegador.
